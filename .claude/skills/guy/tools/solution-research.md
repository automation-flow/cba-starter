# Solution Research (a tool)

A tool, not an agent. It answers one question: has someone already solved this, and
which pattern fits, so we don't reinvent the wheel.

Guy (the orchestrator) uses it after discovery and before pricing, to shape the scope.

## Use it to
- Search the Brain (`wiki/`) for anything close we have built or written before.
- Name the closest known approach or prior build.
- Decide what to reuse versus build new, and what is explicitly out of scope.

## Return
- **Closest prior art / pattern:** ... (from the Brain or a known approach)
- **Reuse vs build:** ...
- **In scope:** ...
- **Out of scope:** ...

A tool stays focused: it informs the scope. It does not run discovery and it does not
set the price.
